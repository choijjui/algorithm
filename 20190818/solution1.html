1/3 숫자 세기

시간 제한 : 2초 메모리 제한 : 256MB

문제 설명

당신의 귀여운 어린 조카는 손가락으로 숫자를 센다. 숫자를 셀때 엄지손가락 부터 시작해서 새끼 손가락까지 접어가면서 숫자를 세는데 새끼손가락을 접고나면 다시 새끼손가락 부터 반대로 손가락을 하나씩 펴는 방식으로 숫자를 센다. 당신의 조카는 집념이 강하기 때문에 원하는 숫자를 셀때까지 멈추지 않는다. 하지만 당신의 조카가 손가락 한개를 다쳤다. 그래서 제한된 횟수 int maxCount 횟수만 해당 손가락을 사용할 수 있다. 엄지부터 새끼손가락을 1부터 5로 표현했을때, int weakFinger는 아픈 손가락을 나타낸다.
당신의 조카가 셀수 있는 가장 큰 숫자를 리턴하시오.

참고 / 제약 사항

  • 1 <= weakFinger <= 5 0 <= maxCount <= 100,000

테스트 케이스

  1. int weakFinger = 2
    int maxCount = 3
    리턴(정답): 15

    처음 15개의 숫자는 1,2,3,4,5,4,3,2,1,2,3,4,5,4,3 이렇게 셀수 있다. 다음은 2번째 손가락을 사용할 차례이지만 이미 3번의 횟수를 모두 사용했기 때문에 더이상 사용할 수 없다.

  2. int weakFinger = 1
    int maxCount = 0
    리턴(정답): 0

    숫자를 셀때 엄지손가락을 사용해야하는데 사용할 수 없으므로 숫자를 셀수 없다. 정답은 0이다.

  3. int weakFinger = 5
    int maxCount = 0
    리턴(정답): 4

    새끼손가락을 전혀사용할 수 없기때문에 4까지 셀수있다.

  4. int weakFinger = 2
    int maxCount = 48
    리턴(정답): 193
  5. int weakFinger = 5
    int maxCount = 973
    리턴(정답): 7788 0 —>
